# leonld94.github.io  
This git's function => json test & create blog page => image cloud website develop  
  
1. 제목 :  Hyper Image Cloud or Private AK-12 Library (임시)  
  
2. 목표 : 태그를 이용한 효과적이고 보기 좋은 이미지 저장 & 정리  
  
3. 기능  
    3_1 작가 분류 : Eng / 中 / 日 / 한 4개 국어의 작가 닉네임 분류  
    3_2 수위 조절 : 순한 맛 / 매콤한 맛 / 매운 맛 / 모두 보여지는 이미지의 수위를 조절 할 수 있음  
    3_3 태그 설정 : 캐릭터 태그, 특수 태그(배포 가능 등) 등의 태그를 붙여 간편하게 검색 가능  
    3_4 업로드 기능 : 이미지를 업로드할 때 이미지의 세부 사항을 조정하여 업로드 가능  
    3_5 다운로드 기능 : 원본 보기 버튼과 다운로드 버튼을 따로 만들어 사용 가능  
    
  
4. 구현에 관한 상세한 아이디어  
    4_1 이미지 미리보기 크기 조절 : 지정된 이미지 크기를 w,h라고 할 때 이미지의 가로 세로를 w/h로 나눈 값을 dw/dh라고 하고 더 크기가 큰 값을 기준으로 가로/세로를 비율에 맞게 축소, 미리보기 이미지로 띄운다.  
    4_2 만화 미리보기 설정 1안 : 그냥 첫번째 이미지를 무식하게 띄운다.  
        만화 미리보기 설정 2안 : 임의의 이미지를 설정하도록 하고 이미지 크기 조절을 따른다.  
    4_3 만화 제공 설정 : imgur 처럼 아래로 내리면 연속되게 만든다.  
    4_4 스위치 설정 : (순한맛 / 매콤한맛 / 매운맛) * (이미지 / 만화) 총 6구역으로 나눠야 할것으로 보인다.  
    4_5 이미지 설정 1안 : json 파일 안에 설정 및 이미지 경로를 넣어둔다.
        이미지 설정 2안 : 업로드 될 때 마다 새로운 폴더를 만들어 그 안에 json 파일로 설정을 저장한다. 
    4_6 업로드 방식 : 업로드 페이지를 따로 만들어 이미지의 세부 설정을 하고 업로드 버튼을 누르면 된다.  
    4_7 정렬 기준 : 기본적인 이미지의 정렬이 필요하다. 아무것도 건들지 않으면 업로드 순서대로 정렬 될 것이다. 사진을 업로드할 때, 대표로 넣은 캐릭터의 이름으로 파일 명을 정하면 대표 캐릭터 순으로 정렬 될 것이다. 이는 태그 추가와 함께 추가적으로 구현해야한다.  
    4_8 업로드 페이지 : 업로드 페이지 안에 일러스트 / 만화 버튼을 만들고 일러스트의 경우 하나씩만 선택 가능하도록, 만화는 여러장 선택 가능하도록 할 예정이다.  
    4_9 작가 정리 : 작가 4개국어 닉네임과 pixiv 링크등을 저장하려면 작가마다 json을 만드는게 좋을 것으로 보인다. 그렇다면 만드는 김에 comic과 illustration 목록도 만들어서 같이 저장하는게 좋을 것이다.  

5. 파일 구조 정리  
  
    5_1 일러스트레이션 저장 방식 1안 : 이미지를 업로드할 때 태그로 정한 대표 캐릭터의 이름이 파일의 이름이 되고 (ex)UMP45_3.png), 동명의 json 파일에 업로드한 상세 내용을 저장, 요구할때마다 설정에서 뜯어본다.  
        일러스트레이션 저장 방식 2안 : 순한 맛 / 매콤한 맛 / 매운 맛 3개의 json 파일을 만들어두고, 해당 일러스트가 추가될 때마다 목록에 추가한다. 나머지는 1안과 같다.   
    5_2 만화 저장 방식 : 작가 이름을 제목으로 설정하고 장 수마다 숫자가 올라간다(ex)abc_1_1.png). 뒤에 숫자를 뺸 제목의 json 파일에 만화 한 세트의 목록과 설정이 들어가고, 순한 맛 / 매콤한 맛 / 매운 맛 json 목록 파일에는 해당 json 파일이 들어간다(ex)abc_1.json).

6. 구현 정리  
    6_1 일러스트 업로드:  
        1. 사진을 업로드 받고, 작가, 등장 캐릭터, 수위 정보를 입력 받는다.
        2. 대표 캐릭터의 이름과 숫자로 이미지 이름을 정하고 저장한다.  
        3. 동일 이름으로 json 파일을 만들고 설정을 저장한다.  
        4. 순함,매콤함,매움 파일 중 해당하는 곳의 목록에 json 파일명을 추가한다.  
        5. 해당 작가의 json 파일의 작품 목록에 json 파일명을 추가한다.  
        6. 대표 캐릭터의 카운트를 올리고 등장 캐릭터들의 일러스트 목록에 올린다(맨 앞에 이름을 올리고 있는 캐릭터가 대표 캐릭터로 자동 설정된다).  
    6_2 일러스트에서 캐릭터 탐색:  
        1. 태그 클릭을 js가 인식, /library/character/ 에서 해당 캐릭터의 json을 열고 illust 목록을 가져온다.
        2. illust 목록을 돌며 해당 일러스트 수위를 확인하고, 해당하면 화면에 띄운다.
