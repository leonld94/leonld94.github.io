# leonld94.github.io  
This git's function => json test & create blog page => image cloud website develop  
  
1. 제목 :  Hyper Image Cloud (임시)  
  
2. 목표 : 태그를 이용한 효과적이고 보기 좋은 이미지 저장 & 정리  
  
3. 기능  
    3_1 작가 분류 : Eng / 中 / 日 / 한 4개 국어의 작가 닉네임 분류  
    3_2 수위 조절 : 순한 맛 / 매콤한 맛 / 매운 맛 / 모두 보여지는 이미지의 수위를 조절 할 수 있음  
    3_3 태그 설정 : 캐릭터 태그, 특수 태그(배포 가능 등) 등의 태그를 붙여 간편하게 검색 가능  
    3_4 업로드 기능 : 이미지를 업로드할 때 이미지의 세부 사항을 조정하여 업로드 가능  
    3_5 다운로드 기능 : 원본 보기 버튼과 다운로드 버튼을 따로 만들어 사용 가능  
    
  
4. 구현에 관한 상세한 아이디어  
    4_1 이미지 크기 조절 : 지정된 이미지 크기를 w,h라고 할 때 이미지의 가로 세로를 w/h로 나눈 값을 dw/dh라고 하고 더 크기가 큰 값을 기준으로 가로/세로를 비율에 맞게 축소, 미리보기 이미지로 띄운다.  
    4_2 만화 미리보기 설정 1안 : 그냥 첫번째 이미지를 무식하게 띄운다.  
        만화 미리보기 설정 2안 : 임의의 이미지를 설정하도록 하고 이미지 크기 조절을 따른다.  
    4_3 만화 제공 설정 : imgur 처럼 아래로 내리면 연속되게 만든다.  
    4_4 스위치 설정 : (순한맛 / 매콤한맛 / 매운맛) * (이미지 / 만화) 총 6구역으로 나눠야 할것으로 보인다.  
    4_5 이미지 설정 1안 : json 파일 안에 설정 및 이미지 경로를 넣어둔다.
        이미지 설정 2안 : 업로드 될 때 마다 새로운 폴더를 만들어 그 안에 json 파일로 설정을 저장한다. 
    4_6 업로드 방식 : 업로드 페이지를 따로 만들어 이미지의 세부 설정을 하고 업로드 버튼을 누르면 된다.  
    4_7 정렬 기준 : 기본적인 이미지의 정렬이 필요하다. 아무것도 건들지 않으면 업로드 순서대로 정렬 될 것이다. 사진을 업로드할 때, 대표로 넣은 캐릭터의 이름으로 파일 명을 정하면 대표 캐릭터 순으로 정렬 될 것이다. 이는 태그 추가와 함께 추가적으로 구현해야한다.  

5. 파일 구조 정리  
  
    5_1 일러스트레이션 저장 방식 1안 : 이미지를 업로드할 때 태그로 정한 대표 캐릭터의 이름이 파일의 이름이 되고 (ex)UMP45_3.png), 동명의 json 파일에 업로드한 상세 내용을 저장, 요구할때마다 설정에서 뜯어본다.
        일러스트레이션 저장 방식 2안 : 순한 맛 / 매콤한 맛 / 매운 맛 3개의 json 파일을 만들어두고, 해당 일러스트가 추가될 때마다 목록에 추가한다. 작가도 마찬가지, 캐릭터도 마찬가지. 단 목록이 작가 수 + 캐릭터 수만큼 늘어난다.
